Scenario Buscar producto scenario

NameBuscar producto scenario
DescriptionEste escenario ocurre cuando un usuario hace una petición de búsqueda al sistema. El objetivo del sistema es mostrar al usuario los productos que coinciden con su petición. La percepción de este escenario es la petición de búsqueda del usuario.
PriorityNot Specified
Actors 
Initiated bySystem
Trigger 
Steps
#TypeNameRoleDescriptionData usedData produced
1PerceptPetición de búsqueda El cliente envía sus criterios de búsqueda  
2GoalFiltrar productos Se ha de filtrar los productos según los criteriosBD Productos 
3GoalGuardar filtros de búsqueda Se deben guardar los filtros de búsqueda del cliente BD Búsquedas
4GoalMostrar resultado búsqueda Se deben devolver los productos que coinciden con los criterios de búsqueda  
5ActionAc. Mostrar resultado búsqueda Se muestran los productos resultantes de la búsqueda  
Variation 


Scenario Venta productos scenario

NameVenta productos scenario
DescriptionEl escenario de venta de productos se produce cuando un usuario quiere comprar uno o más productos del sistema. La percepción de este escenario es la petición de compra por parte del usuario. El objetivo principal del sistema es vender los productos.
PriorityNot Specified
Actors 
Initiated bySystem
Trigger 
Steps
#TypeNameRoleDescriptionData usedData produced
1PerceptLista de compra Se recibe la lista de productos que el cliente desea  
2GoalObtener información de pago y envío Se debe obtener la información de pago y de envío del cliente  
3ActionAc. Pedir info metodo pago y envio Se pide la información de pago y envío al cliente BD Clientes
4PerceptSelección método pago y envío El cliente informa de sus datos e información de pago y envío  
5GoalEnviar factura Se debe enviar la factura al cliente  
6ActionAc. Enviar factura Se envía la factura al clienteBD Clientes 
7GoalGuardar historial de compra Se debe guardar el historial de compra del cliente  BD Compras
8GoalEnviar producto Se deben enviar los productos vendidos  
9ScenarioEnviar productos scenario Se escoge el/los centro/s logístico/s desde donde se envían los productos y se envían   
10PerceptConfirmación envio Se recibe confirmación de que el o los productos han sido enviados  
11GoalRealizar cobro Se debe realizar el cobro al cliente del importe de los productos  
12ActionAc. Solicitar cobro Se cobra al cliente el importe de los productosBD Clientes 
VariationSi uno (o más) de los productos que desea el cliente son de un vendedor externo que se encarga del envío, se deberá notificar de que envíe el producto. Si hay productos externos en la lista de compra, el importe de éstos se deberá pagar al vendedor externo. Este pago se realizará inmediatamente tras la venta de si el vendedor externo se encarga del envío, o posterior a la confirmación de envío si se encarga la tienda.


Scenario Devolver producto scenario

NameDevolver producto scenario
DescriptionEn este escenario se trata la devolución de un producto por parte de un cliente y su correspondiente devolución del dinero por parte del sistema. La percepción de este escenario es la petición de devolución del producto que contiene la razón de la devolución. El sistema procesa la petición para decidir si se acepta o no y notifica al usuario la información necesaria para que pueda devolver el producto.

Initiated bySystem

4ActionAc. Pedir recoger devolución Se informa al transportista encargado de recoger devoluciones del paquete y dirección del clienteTransportistas

6ActionAc. Informar cliente sobre devolución Se informa al cliente sobre la fecha de recogida del paqueteBD Clientes

8GoalRegistrar devolución Se debe registrar la información de la devolución BD Compras

Scenario Enviar productos scenario

NameEnviar productos scenario
Description Este escenario trata el proceso de elección de los centros logísticos que se encargaran de gestionar el envío de los productos comprados por el usuario, al final de este proceso se notificará al usuario si se ha escogido más de un centro logístico. A continuación, cuando el envío de un producto ya ha sido asignado a un centro logístico y es este centro y en este escenario donde se da la negociación con los transportistas.
PriorityNot Specified
Actors 
Initiated bySystem
Trigger 
Steps
#TypeNameRoleDescriptionData usedData produced
1GoalElegir centros logísticos Se debe encontrar el centro logístico para enviar los productos  
2GoalComprobar stock productos Se comprueba que haya stock de los productos solicitados y se modifica su cantidad. Se añaden los productos a enviar a productos pendientesProductos en stockProductos pendientes, Productos en stock
3GoalNotificar info centro logístico Se debe notificar si se envían los productos desde más de un centro logísticoBD Clientes 
4GoalOrganizar productos en lotes Se deben organizar los productos a enviar en lotesProductos pendientesLotes pendientes
5GoalEscoger lotes a enviar Se deben escoger los lotes a enviarLotes pendientesLotes pendientes
6GoalNegociar transportista Se debe negociar con los transportistas para que envíen sus ofertas para enviar los lotes  
7ActionAc. Pedir oferta transporte Se envía a las empresas transportistas la información del loteTransportistas 
8PerceptOfertas transporte Recibimos las ofertas de las empresas transportistas  
9GoalEscoger transportista Se debe escoger el transportista con la mejor oferta  
10ActionAc. Pedir envío lote Pedimos que el transportista envíe el lote  
11GoalRegistrar envío Se debe registrar la información del envío BD Envios
12GoalNotificar sobre entrega Se debe notificar al usuario de la fecha de entrega final y del transportista que realiza la entrega  
13ActionAc. Notificar sobre entrega Se notifica al cliente sobre la fecha de entrega y el transportista que la realizaBD Clientes 
VariationSi en la respuesta de la petición de envío encontramos que el centro logístico no tiene un producto, deberemos buscar otro centro logístico enviándole la petición de envío informando solo de los productos que el otro centro no tenga disponibles. Así hasta que todos los productos se envíen desde un centro logístico.
